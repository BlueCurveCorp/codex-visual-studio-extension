# CFP-003: Auto Context and TODO CodeLens

Priority: P2  
Status: Backlog  
Owner: AI agent  
Depends on: `CFP-001` for `/auto-context` command integration

## Goal

Add automatic IDE context collection and TODO-oriented quick actions so Codex can use active Visual Studio state with shorter prompts.

## User Value

Users should be able to ask "fix this", "explain this error", or "implement this TODO" and have Codex understand the relevant file, selection, open documents, diagnostics, and solution context.

## Functional Requirements

### Auto Context

1. Add an `AutoContextEnabled` setting.
2. Add a UI toggle near the composer and wire `/auto-context`.
3. When enabled, include a compact context block with:
   - active document path
   - current selection range and selected text excerpt
   - caret line
   - open document paths
   - startup project when available
   - solution directory
   - current branch when available
   - relevant diagnostics/errors if accessible
4. Keep context bounded and predictable.
5. Show what context will be sent before sending, or provide a compact disclosure row.
6. Allow user to disable Auto Context per prompt.
7. Do not include secrets from environment variables or settings.

### TODO CodeLens / Quick Actions

1. Detect TODO-like comments in C#, XAML, project files, and common text/code files.
2. Add a Visual Studio command or CodeLens-style action:
   - "Implement with Codex"
   - "Explain TODO"
   - "Write tests for TODO"
3. Action opens/focuses Codex tool window.
4. Action populates or sends a prompt containing:
   - file path
   - line number
   - TODO text
   - surrounding code excerpt
5. The prompt should use `@file` references where possible instead of pasting large files.

## Acceptance Criteria

- User can toggle Auto Context.
- Auto Context state persists across sessions.
- Sending a prompt with Auto Context includes active document and selected text summary.
- Context is bounded and visible to the user.
- `/auto-context` toggles the feature.
- User can invoke "Implement with Codex" from a TODO comment.
- TODO prompt includes file path, line, TODO text, and surrounding code.
- No hardcoded colors or unlocalized text.

## Risks

- Visual Studio CodeLens APIs may require additional extension components. If that is too large, ship context-menu quick actions first.
- Auto Context can leak too much code. Keep strict max lengths and visible disclosure.
- Diagnostics APIs may vary by project system. Treat diagnostics as optional.
