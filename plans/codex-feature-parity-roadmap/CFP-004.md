# CFP-004: Cloud and Worktree Delegation

Priority: P3  
Status: Backlog  
Owner: AI agent  
Depends on: `CFP-001` for `/cloud` and `/local` command integration

## Goal

Add run-mode support for Local, Worktree, and Cloud so users can offload or isolate Codex tasks from Visual Studio.

## User Value

Some tasks are too large, risky, or parallel for the current checkout. Cloud delegation and worktree execution let users keep working while Codex handles larger or isolated jobs.

## Functional Requirements

### Run Modes

Add a run mode selector with:

- Local
- Worktree
- Cloud

Local remains the default and must preserve existing behavior.

### Worktree Mode

1. Detect whether the working directory is inside a Git repository.
2. Let the user create a worktree for a new Codex task.
3. Generate a branch/worktree name with a safe prefix such as `codex/`.
4. Let the user override name before creation.
5. Create worktree using `git worktree add`.
6. Start Codex in the worktree path.
7. Show clear status and path.
8. Provide cleanup guidance, but do not delete worktrees automatically without confirmation.
9. Thread list should display run mode and path when available.

### Cloud Mode

1. Detect whether installed Codex CLI supports cloud delegation.
2. Detect or list configured cloud environments if supported.
3. Let user choose a cloud environment.
4. Let user choose local changes behavior:
   - run from main/base
   - include local changes when supported
5. Start a cloud task through supported CLI/app/deep-link flow.
6. Show task ID/status if returned.
7. Provide "Open in Codex app/web" action if local status tracking is not available.
8. Degrade gracefully when cloud access is unavailable.

### Mode Commands

Wire:

- `/local` to Local mode
- `/cloud` to Cloud mode

Optionally add `/worktree` even though it was not in the initial command list.

## Architecture Plan

### Models

Add:

- `CodexRunMode`
- `CodexCloudEnvironmentSummary`
- `CodexWorktreeSummary`
- `CodexDelegatedTaskSummary`

### Services

Add or extend:

- `GitWorktreeService`
- `CodexCloudService`
- `CodexCapabilityService`

`CodexCapabilityService` should probe installed CLI support and cache results:

- CLI version
- supported subcommands
- support for JSON output
- cloud capability
- worktree-related behavior, if any

### Settings

Persist:

- `DefaultRunMode`
- `PreferredCloudEnvironment`
- `WorktreeRootOverride` if needed
- `WorktreeBranchPrefix`

Do not persist cloud credentials. Use Codex CLI/app auth state.

## Acceptance Criteria

- Existing local execution remains unchanged by default.
- User can select Local mode and run a task.
- User can select Worktree mode in a Git repo and create an isolated worktree.
- Worktree task starts in the worktree directory.
- User sees worktree path and branch.
- `/local` switches to Local mode.
- `/cloud` switches to Cloud mode or shows a useful unavailable message.
- Cloud mode does not fail silently when unsupported.
- No credentials are stored by the extension.

## Risks

- Codex cloud CLI contracts may drift. Probe capabilities and degrade gracefully.
- Worktree creation modifies Git repository state. Validate paths and ask for confirmation before creation.
- Local changes can complicate worktree creation. Warn if current branch has uncommitted changes and explain whether they are included.
- Cloud task status may not be available locally. Prefer app/web deep links when polling is unsupported.
